# Code of Conduct

This project follows a simple standard: be respectful, specific, and constructive.

## Expected Behavior

- Use welcoming and inclusive language.
- Keep technical criticism focused on the work, not the person.
- Respect privacy around recipe data, household data, tokens, logs, and deployment details.
- Report security issues through the private process described in `SECURITY.md`.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Publishing private tokens, logs, recipe exports, or deployment details without consent.
- Using issue reports to disclose exploitable security details publicly.

Maintainers may moderate issues, discussions, or pull requests that do not follow these rules.
